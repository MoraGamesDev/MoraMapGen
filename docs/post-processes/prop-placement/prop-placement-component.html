    Prop Placement Component

    Setting up the Prop Placement component

    Add this component to the same GameObject that has your MoraMapGen component. This is what it looks like by default:

    Prop Density Multiplier

    Controls the overall density of props across all Area Definitions and Tile Module Definitions. A value of 1 means all PropSpots and ProceduralPropSpots will be used in attempting prop placement. A value of 0 means no props will be placed except the minimum required ones.

    Overlap Tolerance

    The amount of acceptable overlap when placing props. When an attempt is made to place a prop, it is checked for overlap between any of its colliders and those of other objects already in the scene. If the overlap distance is greater than this tolerance, its placement is cancelled, and a different prop may be attempted to be placed in the same spot.

    Procedural Offset Defaults

    Optional Procedural Prop Spot Offsets asset for determining possible placement deviation when placing props on Procedural Prop Spots. If no asset is assigned, these props will not have deviation to their positioning.

    Show Logs

    Allow optional debug logs in the console.
