    Create Your Own MapGen Post-Processes

    If you need something to happen before the map is considered “ready”, you can have your scripts implement the IMapGenPostProcess interface. A MapGen Post-Process component will automatically be triggered to do its thing after the map is Baked (tile placement is set) but before the map is considered “ready”, ie, before OnBakedMapReady gets called.

    A Post-Process is assigned a PostProcessPhase to further specify at one point during the post-process step it should apply. The options are BeforeProps, Props, AfterProps, and AfterNavMesh.

    The sequence of events is:

    1. Map gets baked
    2. BeforeProps post-processes are triggered
    3. Props post-processes are triggered (such as the Prop Placement component)
    4. AfterProps post-processes are triggered
    5. Nav Mesh gets baked
    6. AfterNavMesh post-processes are triggered

    IMapGenPostProcess interface

    OnMapBaked(BakedMap bakedMap)

    Is called by MoraMapGen when the map is baked. Passes the resulting BakedMap object.

    Reset()

    Is called by MoraMapGen when the map is cleared. Used to reset the state of the scene so that it’s ready for a fresh new map.

    GetPhase()

    Returns the PostProcesPhase that this post-process should use.
