    Tile Module Definitions

    A Tile Module Definitions asset defines which prefabs are used to construct the tiles that make up the map. See Create Module prefabs for more info.

    For each of the module prefab categories, you can specify as many prefabs as you want, and they will be randomly selected when spawning objects. This adds visual variation to your maps.

    Creating a Tile Module Definitions asset

    To create a new Tile Module Definitions asset, right click in your Project Assets and select “Create/MoraMapGen/Tile Module Definition.”

    Defining the Tile Module Definitions

    Generally you will want at least 1 prefab assigned to each category, but they aren’t required.

    Floors

    Specify the ground prefabs.

    Rooms and Corridors each use two different floors:

    Floor A

    Used where the sprite templates have room pixels (red by default) and wall pixels (blue by default). Eg: Main floor in rooms, and under walls in rooms and corridors.

    Floor B

    Used where the sprite templates have corridor pixels (black by default), Eg: Main paths in corridors and the entrances of rooms.

    Walls Straight

    Specify the straight wall prefabs. Used for straight wall segments.

    Walls Corner Inside

    Specify the inside corner wall prefabs. These are the convex corners, like when rounding a corner in a hallway.

    Walls Corner Outside

    Specify the outside corner wall prefabs. These are the concave corners, like the corner of a room.

    Exterior Ceiling

    Specify the exterior ceiling prefabs. These are visible from above and outside the rooms/corridors. Used when "Use Exterior Ceiling" is enabled.

    Interior Ceiling

    Specify the interior ceiling prefabs. These are visible from inside the rooms/corridors. Used when "Use Interior Ceiling" is enabled.

    Whole Doors

    Specify the door prefabs you want to use when using the Whole Door placement mode. This should include the door frame and the opening/animating and/or logical components of your doors. The entire door (frame + panel) is a single prefab in this mode.

    Half Door Frames

    Specify the prefabs of just one side of the door frame to use when using the Half-Frame door placement mode. This should NOT include the opening portion of the door—just the static frame. This allows each side of the door to use a different style matching the room or corridor it belongs to.

    Half Door Panels

    This is the prefab for the opening portion of your doors for use when using the Half-Frame door placement mode. This includes the actual door panel(s) and any opening/animation logic.

    Prop Categories

    Tile Module-wide prop placement settings. Use this to define general props that should appear with these tile modules across all rooms and corridors that use them. See Prop Categories for descriptions of each category.
